Visão Geral

Sobre o Sistema

O NEP Delivery Control é uma solução proprietária desenvolvida para eliminar a "caixa preta" operacional em ambientes de Qualidade, oferecendo rastreabilidade total desde o micro (monitoria individual) até o macro (visão executiva).

Objetivo do Projeto

Criar um sistema zero-custo, altamente customizável e específico para a dor de Governança de Qualidade, substituindo soluções genéricas (Monday/Jira/Trello) que custam R$ 120k/ano e não possuem funcionalidades críticas como:

  • Hierarquia Granular Nativa: 6 níveis de acesso (0-5) com permissões específicas
  • Trava de Evidência Obrigatória: Bloqueio automático de entrega sem anexo
  • Validação/Aprovação: Fluxo de auditoria pós-entrega
  • Reset Automático: Rotinas diárias recriadas automaticamente
  • Visão Holística: Do operacional ao estratégico em um único painel
Problema Resolvido

Antes: Demandas espalhadas por e-mails, WhatsApp, planilhas offline sem dono, zero rastreabilidade, impossibilidade de gerar relatórios executivos.

Depois: Backlog unificado, status em tempo real, auditabilidade total, BI automático e compliance operacional garantido.

40+ Colaboradores

Gerenciamento simultâneo de Consultores, Analistas, Coordenadores e Monitores com permissões granulares.

100% Auditável

Custo Zero

Visualização analítica com gráficos dinâmicos:
• Status das Demandas (Doughnut Chart)
• Demandas por Solicitante (Bar Chart)
• Demandas por Unidade (Horizontal Bar)

Geração de Charts JavaScript 
function updateCharts() {
  const visible = getVisibleTasks();
  
  const statusData = [0, 0, 0, 0];
  visible.forEach(t => {
    if (t.status === 'backlog') statusData[0]++;
    if (t.status === 'doing') statusData[1]++;
    if (t.status === 'pending') statusData[2]++;
    if (t.status === 'done') statusData[3]++;
  });

  new Chart(ctx, {
    type: 'doughnut',
    data: {
      labels: ['Backlog', 'Execução', 'Block', 'Entregue'],
      datasets: [{ 
        data: statusData, 
        backgroundColor: ['#2F6FED', '#F2B705', '#E5533D', '#2ECC71']
      }]
    }
  });
}

• Banner "Nova demanda de [Cargo] - [Nome]"

Lógica de Notificação JavaScript 
function countUnacknowledgedTasks() {
  const myTasks = allTasks.filter(t => {
    const assignedToMe = myName && 
      norm(t?.owner) === norm(myName);
    const isNew = t.status === 'backlog' && 
      !t.acknowledged;
    return assignedToMe && isNew;
  });
  return myTasks.length;
}

function updateNotificationBadge() {
  const count = countUnacknowledgedTasks();
  const badge = document.getElementById('notification-badge');
  
  if (count > 0) {
    badge.textContent = count;
    badge.style.display = 'flex';
  } else {
    badge.style.display = 'none';
  }
}

7. SLA Timer Visual

Contador regressivo em cada card baseado no campo deadline:
• Verde: Mais de 2 dias restantes
• Amarelo: Entre 4h e 2 dias
• Vermelho: Menos de 4h ou expirado

Timer Automático JavaScript 
setInterval(() => {
  document.querySelectorAll('.card').forEach(card => {
    const task = allTasks.find(t => t.id === card.dataset.id);
    if (!task || !task.deadline) return;

    const deadline = new Date(task.deadline + 'T23:59:59');
    const diff = deadline - new Date();

    if (diff < 0) {
      slaDiv.className = 'sla-timer sla-danger';
      slaDiv.innerHTML = '⚠️ EXPIRADO';
    } else {
      const hours = Math.floor(diff / (1000 * 60 * 60));
      slaDiv.className = hours < 4 ? 
        'sla-timer sla-danger' : 'sla-timer sla-warn';
      slaDiv.innerHTML = `⏱️ ${hours}h`;
    }
  });
}, 1000);

8. Sistema de Filtros Avançados

Filtros dinâmicos na sidebar que atualizam automaticamente os selects baseados nas demandas visíveis:

  • Por Unidade: Comercial, Operações, TI, etc.
  • Por Segmento: Desenvolvimento, Monitoria, Auditoria, etc.
  • Por Solicitante: Nome de quem criou a demanda
  • Por Responsável: Nome do executor atribuído

Todos os filtros funcionam em conjunto (AND lógico) e aplicam-se tanto ao Kanban quanto ao Dashboard BI.

Limitação Conhecida

O sistema não utiliza localStorage/sessionStorage para persistência de dados das demandas (apenas para Rotina ADM). Toda informação de tasks está no Firestore e sincroniza em tempo real via listeners.

Implementação de Permissões

Funções de Controle JavaScript 
function canSeeTask(task) {
  const tLevel = taskCreatorLevel(task);
  
  // Superintendente vê tudo
  if (myRoleKey === 'superintendente') return true;
  
  // Se EU sou o responsável, SEMPRE vejo
  const assignedToMe = myName && 
    norm(task?.owner) === norm(myName);
  if (assignedToMe) return true;
  
  // Criador sempre vê suas tarefas
  if (isCreatorMe(task)) return true;
  
  // Monitor só vê tarefas de monitor
  if (myRoleKey === 'monitor') {
    return tLevel === 0;
  }
  
  // Demais veem nível igual ou inferior
  return tLevel <= myLevel;
}
